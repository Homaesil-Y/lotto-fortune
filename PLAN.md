# 로또 분석기 — 기획서 (v1)

> 작성일: 2026-06-23 · 상태: 기획 확정, 골격 작업 시작

## 1. 목적 (Purpose)
과거 로또 당첨 데이터를 **재미있게 탐색**하고, 통계 패턴을 근거로 **번호 조합을 추천**받는 웹 도구.
핵심 가치는 *예측*이 아니라 **"재미있는 탐색 + 그럴듯한 추천"**.

## 2. 컨셉 (Concept)
두 축을 한 화면 흐름으로 연결한다: **[보기] 통계 시각화 → [고르기] 통계 기반 추천**

- 축 1 — 통계 시각화(탐색): 출현 빈도, 홀짝/고저/구간 분포, 미출현 기간
- 축 2 — 통계 기반 추천(생성): 빈도·미출현·균형·랜덤 모드, 추천 근거를 함께 표시

### 전제 (꼭 명시)
로또는 **독립 시행**이라 어떤 추천도 당첨 확률을 높이지 않는다.
→ "예측기"가 아니라 "탐색·재미 도구"로 포지셔닝. 추천 영역에 가벼운 면책 한 줄.

### 타깃
**재미/오락용 개인 프로젝트.** 인터랙션·디자인·"오 신기하다" 순간에 투자.
사용자 계정 / SEO / 무거운 운영 자동화는 범위 제외.

## 3. 데이터 소스 (검증 완료)
- 엔드포인트: `https://www.dhlottery.co.kr/common.do?method=getLottoNumber&drwNo={회차}`
- 인증 불필요, 단순 GET. `drwNo`만 1→최신까지 순회하면 전 회차 수집 가능.
- ⚠️ **해외 IP 차단** → 데이터 수집/갱신은 한국 IP 환경에서 실행해야 함. 결과는 정적 `lotto.json`으로 캐싱.

### 응답 스키마
| 필드 | 의미 |
|---|---|
| `returnValue` | 성공 여부 (success/fail) — 미추첨 회차 판별 |
| `drwNo` | 회차 |
| `drwNoDate` | 추첨일 |
| `drwtNo1`~`drwtNo6` | 당첨번호 6개 |
| `bnusNo` | 보너스 번호 |
| `firstWinamnt` / `firstPrzwnerCo` / `firstAccumamnt` | 1등 상금/당첨자수/총상금 |
| `totSellamnt` | 총 판매액 |

## 4. MVP 기능 명세
| # | 기능 | 우선순위 | 상태 |
|---|---|---|---|
| F1 | 최신 당첨번호 헤더 | MVP | ✅ 완료 |
| F2 | 번호별 출현 빈도 차트 | MVP | ✅ 완료 (lazy 청크) |
| F3 | 미출현 기간 | MVP | ✅ 완료 |
| F4 | 패턴 분포 (홀짝/고저/구간) | MVP | ✅ 완료 |
| F5 | 번호 추천 (빈도/미출현/균형/랜덤 모드 + 근거) | MVP | ✅ 완료 |
| F6 | 회차 검색 | MVP | ✅ 완료 |
| F7 | 추천 복사/저장 (localStorage, 최근 10개) | 2차 | ✅ 완료 |
| F8 | 연속번호·끝자리 분석 | 2차 | ✅ 완료 |

> 남은 작업: 실데이터 수집(한국 환경 `npm run fetch`) 뿐. 코드 기능은 MVP+2차 모두 구현·검증 완료.

### 추천 모드 (F5)
- 🔥 빈도 가중 · ❄️ 미출현 보정 · ⚖️ 균형(홀짝·구간 평균 맞춤) · 🎲 완전 랜덤(정직한 대조군)

## 5. 화면 구조
단일 대시보드, 세로 흐름 = 시선 흐름:
`F1 최신번호 → F2 빈도 / F4 패턴 → F3 미출현 → F5 추천(색 배경으로 분리)`
추천 번호는 항상 "근거 한 줄"과 함께 표시. 모바일은 2단 그리드 → 1단 반응형.

## 6. 기술 스택 / 구조
- Vite + React + TypeScript, 차트는 Recharts, 정적 `public/lotto.json`
- 데이터 수집: `scripts/fetch-lotto.mjs` (Node, 증분 갱신 지원) — 한국 환경에서 실행
- 배포: 정적 호스팅(프론트). 주간 갱신은 한국 리전에서 스크립트 실행 후 재배포

## 7. 범위 밖 (Out of scope, v1)
